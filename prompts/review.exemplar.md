# Review — example

For each finding: what's wrong, severity (critical/high/medium), and the
concrete fix. Covers correctness, edge cases, error paths, and security
(e.g. untrusted input crossing a trust boundary). Ends with a clear verdict and
the must-fix items called out, not buried.
