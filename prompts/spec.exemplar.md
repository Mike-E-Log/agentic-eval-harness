# Spec — example

**Problem & premises:** one paragraph each; premises stated, not assumed.

**Architecture:** components + how they connect (a diagram is fine).

**Data model:** the core records and their fields, with a schema version.

**API/CLI contract:** every command/endpoint, its args, and its output shape.

**Error model:** each failure mode -> the state/message the user sees.

**Tests:** the suites and what each guards; failure modes covered.

**Scope / non-goals:** what is explicitly cut from this version.

**Success criteria:** a verifiable checklist.
